## 1. 打开 5 个终端（建议顺序启动）

项目根目录：`d:\desktop\Carbon_manage_system\carbon_official`

## 2. 后端：先安装公共依赖（只需一次）

在 `carbon_back\carbon-frame`：

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_back\carbon-frame
mvn -DskipTests install
```

> 仓库里有自带 Maven：`carbon_back\apache-maven-3.8.8\bin\mvn.cmd`。
> 如果你本机没装 Maven，可用下面这种写法：
>
> ```powershell
> cd d:\desktop\Carbon_manage_system\carbon_official\carbon_back\carbon-frame
> ..\apache-maven-3.8.8\bin\mvn.cmd -DskipTests install
> ```

## 3. 后端：分别启动各服务（每个服务一个终端）

### 3.1 启动认证服务 carbon-auth（端口 9001）

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_back\carbon-auth
mvn -DskipTests spring-boot:run
```

### 3.2 启动系统服务 carbon-system（端口 9002）

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_back\carbon-system
mvn -DskipTests spring-boot:run
```

### 3.3 启动资产服务 carbon-assets（端口 9003）

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_back\carbon-assets
mvn -DskipTests spring-boot:run
```

### 3.4 启动交易服务 carbon-trade（端口 9005，可选）

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_back\carbon-trade
mvn -DskipTests spring-boot:run
```

### 3.5 最后启动网关 carbon-gate（端口 9091，前端默认走它）

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_back\carbon-gate
mvn -DskipTests spring-boot:run
```

## 4. 前端：安装依赖并启动（端口 9528）

在 `carbon_front`：

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_front
npm install
```

然后启动：

```powershell
cd d:\desktop\Carbon_manage_system\carbon_official\carbon_front
$env:NODE_OPTIONS="--openssl-legacy-provider"
npm run dev
```

浏览器访问：
- 前端：`http://localhost:9528/`
- 网关：`http://localhost:9091/`

## 5. 启动成功的快速验证

- 验证网关路由 + 认证服务：打开
  - `http://localhost:9091/authCenter/auth/captcha`

如果这个能返回 `captchaId/imageBase64`，说明网关 + auth 服务已正常。

确认这个流程后，我可以再给你一份“启动失败时如何快速定位是哪一环问题”的排查清单。
